<h2>Compilation of Submissions</h2>
<p>During the contest, teams will submit proposed solutions to the contest problems to the Judges using the <a href="https://www.domjudge.org/">DOMjudge</a> contest control system. This can be done through the <a href="https://www.domjudge.org/docs/manual/8.1/team.html#web-interface">web interface</a>, or using the <a href="https://www.domjudge.org/docs/manual/8.1/team.html#command-line-submit">submit client</a>. Source files submitted to the Judges will be compiled using the following command line arguments for the respective language:</p>
<ul>
<li>
<p>C:</p>
<ul>
<li>gcc -std=gnu17 -x c -Wall -O2 -static -pipe -o $1 &quot;$1.c&quot; -lm</li>
<li>Available as mygcc</li>
</ul>
</li>
<li>
<p>C++:</p>
<ul>
<li>g++ -std=gnu++20 -x c++ -Wall -O2 -static -pipe -o $1 &quot;$1.cpp&quot; -lm</li>
<li>Available as mygpp</li>
</ul>
</li>
<li>
<p>Python 3:</p>
<ul>
<li>pypy3 $@</li>
<li>Available as mypython</li>
</ul>
</li>
<li>
<p>Java:</p>
<ul>
<li>javac -encoding UTF-8 -sourcepath . -d . $@</li>
<li>Available as myjavac</li>
</ul>
</li>
<li>
<p>Kotlin:</p>
<ul>
<li>kotlinc -d . $@</li>
<li>Available as mykotlinc</li>
</ul>
</li>
</ul>
<h2>Execution of Submissions</h2>
<p>For each language, if the above compilation step is successful then the submission will be executed as follows:</p>
<ul>
<li>For C/C++: the executable file generated by the compiler will be executed to generate the output of the submission.</li>
<li>For Python 3: the main source file will be executed by the PyPy3 Python3 interpreter to generate the output of the submission.</li>
<li>For Java: the compiled main class will be executed using the following command:
<ul>
<li>java -Dfile.encoding=UTF-8 -XX:+UseSerialGC -Xss65536k -Xms1966080k -Xmx1966080k $@</li>
<li>Available as myjava</li>
</ul>
</li>
<li>For Kotlin: the compiled main class will be executed using the following command:
<ul>
<li>kotlin -Dfile.encoding=UTF-8 -J-XX:+UseSerialGC -J-Xss65536k -J-Xms1966080k -J-Xmx1966080k $@</li>
<li>Available as mykotlin</li>
</ul>
</li>
</ul>
<p>Compilation and execution as described above will take place in a &quot;sandbox&quot; on a dedicated judging machine. The judging machine will be identical to the machines used by teams (see Hardware above).</p>
<p>The sandbox will allocate 2 GiB of memory; the entire program, including its runtime environment, must execute within this memory limit. For interpreted languages (Python, Java, and Kotlin) the runtime environment includes the interpreter (that is, the PyPy3 interpreter for Python and the JVM for Java/Kotlin). The sandbox memory allocation size will be the same for all languages and all contest problems. For Java and Kotlin, the above commands show the stack size and heap size settings that will be used when the program is run in the sandbox.</p>
